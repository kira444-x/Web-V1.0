input

In HTML, the Input element is used to create interactive controls in web forms that accept user data. It can accept a variety of input types based on the type attribute. Here's a breakdown of the different input types, attributes, and how they work:

1. Basic Syntax

input type="..."

2. Input Types

The input element supports many different input types, each of which creates a different form control.

3. pattern attribute

The pattern attribute specifies a regular expression that the input element's value is checked against on form submission.

Note: The pattern attribute works with the following input types: text, date, search, url, tel, email, and password.

Tip: Use the global title attribute to describe the pattern to help the user.

Character Classes

Character classes allow you to define which characters are allowed in the input

Character Description and Example
\d Any digit (0-9).
Example: pattern="\d+" allows one or more digits.
\D Any non-digit character.
Example: pattern="\D+" allows one or more non-digit characters.
\w Any word character (a-z, A-Z, 0-9, and underscores _).
Example: pattern="\w+" allows one or more word characters.
\W Any non-word character.
Example: pattern="\W+" allows one or more non-word characters.
\s Any whitespace character (spaces, tabs, line breaks).
Example: pattern="\s+" allows one or more whitespace characters.
\S Any non-whitespace character.
Example: pattern="\S+" allows one or more non-whitespace characters.

Quantifiers

Quantifiers specify how many instances of a character or group are required.

Quantifier Description and Example
+ One or more of the preceding element.
Example: \d+ allows one or more digits.
* Zero or more of the preceding element.
Example: \d* allows zero or more digits.
? Zero or one of the preceding element (optional).
Example: \d? allows zero or one digit (optional).
{n} Exactly n instances of the preceding element.
Example: \d{5} allows exactly 5 digits.
{n,} At least n instances of the preceding element.
Example: \d{3,} allows at least 3 digits.
{n,m} Between n and m instances of the preceding element.
Example: \d{2,4} allows between 2 and 4 digits.

Anchors

Anchors are used to specify the position of the pattern in relation to the start or end of the string.

Anchor Description and Example
^ Start of string.
Example: ^abc matches "abc" only if it is at the start of the string.
$ End of string.
Example: abc$ matches "abc" only if it is at the end of the string.
^pattern$ Ensures the entire input matches the pattern (from start to end).
Example: ^\d{5}$ ensures exactly 5 digits and no more.

Groups & Ranges

Groups allow you to group part of a pattern, and ranges let you specify a range of characters.

Expression Description and Example
(abc) Grouping.
Example: (abc)+ allows one or more instances of "abc".
[abc] Character set. Matches any one of the characters inside the brackets.
Example: [aeiou] allows any vowel (a, e, i, o, u).
[^abc] Negated character set. Matches any character except the ones inside the brackets.
Example: [^aeiou] matches any character except vowels.
[a-z] Range. Matches any character in the specified range.
Example: [a-z] matches any lowercase letter.
[A-Z] Uppercase range.
Example: [A-Z] matches any uppercase letter.
[0-9] Digit range.
Example: [0-9] matches any single digit.
[a-zA-Z0-9] Alphanumeric range.
Example: [a-zA-Z0-9] matches any alphanumeric character.

Alternation

Alternation allows you to specify multiple alternatives.

AlternationExmple
|: Or abc|def matches either "abc" or "def".

Literal characters

You can match literal characters like letters, numbers, or symbols.

Literal charactersExmple
Any character (a, b, 1, 2, !, @, etc.) can be used directly. pattern="abc" would match exactly "abc" and nothing else.
Escape special charactersExmple
Use \ to escape special characters if you want them to be interpreted literally.pattern="\." matches a literal period . and pattern="\\\\" matches a literal backslash
